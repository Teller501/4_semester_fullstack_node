URL Anatomy

Protocol

Specifies what protocol that's being used to access this resource. There are different protocols for different purposes:

  • HTTP: Hyper Text Transfer Protocol, how web browsers and servers communicate.
  • HTTPS: Secure HTTP, connection with encryption.
  • FTP: File Transfer Protocol, transfers files between computers on network.
  • TCP/IP: Transmission Control Protocol / Internet Protocol address, foundation of the internet - reliable data delivery.
  • DNS: Domain Name System, translates domain names into ip adresses.

Domain

Points to a location on the internet where this resource is.

Sub-domain

Prefix for seperating different parts of a website

Top-level Domain

A URL suffix, that indicates what type of webiste it is ('.com' for commercial, '.net' for network, '.gov' for government etc.). It's the last part of the domain name.

Endpoint

Endpoints are defined by an API and refer to a specific functionality. In REST API, the endpoints are nouns in plural used to access information of a specific collection, such as '/users' or '/drinks'.

Path

The path to the resouce on the specific domain.

Query String / Parameters

The Query String starts with a '?' followed by one or more query parameters, which are key value pairs. If more than one parameter, they are seperated by a '&'. The query string is used to send data to the server.

Path Variables

Path Variables are part of the path itself, and are a dynamic piece of the path. They are used to pass data to the server.
