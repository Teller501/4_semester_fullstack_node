Node.JS Fundamentals

Node.JS is a JavaScript runtime environment, that is used to make it convenient for having JavaScript in both Frontend and Backend.

Basics

To run files with Node, we use the following command:

            
node -app_name-

To install an external module, run one of the following command:

            
npm install 'module_name'       
npm i 'module_name'

You can also manually get them by naming them in the dependencies object in the package.json 

            
"dependencies": {
    "express": "^4.18.3"
}

package.json

The package.json file is a crucial file when working with Node.js. The file is used to store information on the application, dependencies, scripts and more. In Node.js the most common package managers are npm & yarn. 

            
{
    "name": "teller_mandatory_i",
    "version": "0.0.1",
    "type": "module",
    "description": "the first mandatory assignment for the fullstack node js course on KEA (2024)",
    "main": "app.js",
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1"
    },
    "repository": {
        "type": "git",
        "url": "git+https://github.com/Teller501/4_semester_fullstack_node.git"
    },
    "keywords": [
        "node",
        "documentation",
        "code",
        "express"
    ],
    "author": "Anders Teller",
    "license": "MIT",
    "bugs": {
        "url": "https://github.com/Teller501/4_semester_fullstack_node/issues"
    },
    "homepage": "https://github.com/Teller501/4_semester_fullstack_node#readme",
    "dependencies": {
        "express": "^4.18.3"
    }
}

This is the package.json for this project. It contains a lot of meta data and a dependency, which is express. Here is a further explanation of all the key/value pairs:

  1. "name": the name of the module of the package.json.
  2. "version": the current version of the module, typically following semantic versioning.
  3. type: describe the module format this project uses. "module" allows the use of ECMAScript files.
  4. "description": description of the module, describing more than the name can.
  5. "main": the directory of the entry point to the module.
  6. "scripts": can contain many scripts, where the key is the name of the script and the value is the custom bash script.
  7. "repository": describes what type of version control the module uses, and where the source code can be located.
  8. "keywords": collection of keywords that help describe the module and for searching purposes.
  9. "author": the author of the module.
  10. "license": describes what license this module uses.
  11. "bugs": object that describes where you can report bugs for the project.
  12. "homepage": a link to the project homepage. In this case, the link points to the repository readme file.
  13. "dependencies": is used to describe what other modules this module uses. The key is the name of module, and the value is the version.

node_modules

The node_modules folder is a storage folder for all the external modules that your project has installed / depends on. Whenever you run npm install, the module gets downloaded from the npmjs.com website and placed in the node_modules folder.

Nodemon

Nodemon is a great tool for developers, because it automatically restarts the server whenever a change has been made.

When in same directory as your app.js or entry point, run the following command: 

            
nodemon 'your_app'

This will initially do the same as node 'your_app', but it listens for changes, so it runs that command every time a change has been made to the default extensions (js,mjs,cjs,json).

To set your own extenison, you can either run command nodemon --ext js,html,json, or you can create a 'nodemon.json' file with the following: 

            
{
    "ext": "html,js,json,css"
}

This way you do not have to write the extensions everytime you launch the app.

Note: This tool is only intended for development environments, and shouldn't be used in production.

Import/Export

When working with external modules in Node, you can either use CommonJS and ECMAScript modules.

CommonJS modules are the traditional way of packing JavaScript code for Node. CommonJS modules load synchronously.
ECMAScript modules are the standard way making modules for Node.JS now. They load asynchronously, and have several perfomance benefits.

There is a difference in the way of importing/exporting them:

CommonJS

            
// Importing
const express = require("express");             

// Exporting
module.exports = myFunction;

module.exports = {
    myFunction,
    myVariable
  };

ECMAScript

            
// Importing
import { Router } from "express";

const router = Router();

// Exporting
export default router;

To use ECMAScript modules in your project, you can specify the "type" in package.json with the value "module":
"type": "module"

Scripts

In 'package.json' you can define custom bash commands for your project in the "scripts" object: 

            
"scripts": {
    "tester": "pwd"
}

To run the script, you simply type npm run tester in the terminal, and for this script you will get the output:
